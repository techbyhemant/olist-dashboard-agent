# Compact schema summary (for the LLM system prompt)

This is the condensed contract handed to the model. Tables are DuckDB views over the CSVs (query in place). Revenue = SUM(order_items.price) at item grain. Filter order_status = 'delivered' for completed-business KPIs.

```text
orders(order_id PK, customer_id FK->customers, order_status, order_purchase_timestamp,
       order_approved_at, order_delivered_carrier_date, order_delivered_customer_date,
       order_estimated_delivery_date)  -- TIMESTAMPs; status: delivered|shipped|canceled|
       unavailable|invoiced|processing|created|approved
order_items(order_id FK->orders, order_item_id, product_id FK->products, seller_id,
       shipping_limit_date, price DOUBLE, freight_value DOUBLE)  -- order_item_id is a
       per-order line number (1..n), NOT a global id; up to 21 lines/order
products(product_id PK, product_category_name [Portuguese, may be NULL], product_*_lenght,
       product_photos_qty, product_weight_g, product_length/height/width_cm)
customers(customer_id PK, customer_unique_id [person across orders], customer_zip_code_prefix,
       customer_city, customer_state)  -- customer_id is 1:1 with an order
payments(order_id FK->orders, payment_sequential, payment_type [credit_card|boleto|voucher|
       debit_card|not_defined], payment_installments, payment_value DOUBLE)  -- up to 29 rows/order
reviews(review_id, order_id FK->orders, review_score 1..5, review_comment_title/message [often NULL],
       review_creation_date, review_answer_timestamp)  -- ~1/order; 768 orders have none
category_translation(product_category_name [Portuguese] -> product_category_name_english)
```

Gotchas the SQL must respect:
- order_items / payments have MULTIPLE rows per order → aggregate before joining or use COUNT(DISTINCT order_id) to avoid double-counting.
- product_category_name is Portuguese → LEFT JOIN category_translation for English labels (2 categories untranslated, 610 products NULL).
- Not all orders are delivered; delivery timestamps can be NULL → AVG ignores NULLs, but filter status for clean KPIs.
- Time series edges are sparse (2016-09, 2018-09/10) → healthy range is 2017-01 .. 2018-08.
